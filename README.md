# مینی‌اپ پادکست تلگرام — راهنمای کامل از صفر تا راه‌اندازی

نرم‌افزار از قبل کامل ساخته شده؛ این راهنما فقط نحوه‌ی آپلود و وصل‌کردن قطعات به همدیگه‌ست.
هیچ‌جا نیاز به نوشتن کد نیست.

## معماری (همه‌چیز رایگان، بدون سرور شخصی)

| بخش | محل میزبانی |
|---|---|
| مینی‌اپ (رابط کاربری اصلی) | GitHub Pages |
| پنل ادمین | همون GitHub Pages، مسیر `/admin/` |
| دیتابیس اپیزودها | فایل `data/episodes.json` همین ریپازیتوری |
| پخش فایل صوتی | یک Cloudflare Worker رایگان (پروکسی امن، توکن بات رو مخفی نگه می‌داره) |

---

## مرحله ۱ — ساخت ریپازیتوری گیت‌هاب

۱. برو به github.com و اگه حساب نداری Sign up کن (رایگان).
۲. روی + بالای صفحه بزن → **New repository**.
۳. اسم بده (مثلاً `podcast-app`)، گزینه‌ی **Public** رو نگه دار → **Create repository**.

## مرحله ۲ — آپلود فایل‌های پروژه

۱. زیپی که برات فرستادم رو روی کامپیوترت Extract/Unzip کن — یک پوشه به اسم `podcast-miniapp` باز می‌شه.
۲. توی صفحه‌ی خالی ریپو، روی لینک **uploading an existing file** کلیک کن.
۳. تمام فایل‌ها و پوشه‌های **داخل** پوشه `podcast-miniapp` (نه خود پوشه) رو بکش و بنداز توی باکس آپلود.
۴. پایین صفحه **Commit changes** رو بزن.

اگه دراگ‌ودراپ پوشه‌های تو در تو (مثل `src/components`) به مشکل خورد، نرم‌افزار رایگان
[GitHub Desktop](https://desktop.github.com) ساده‌تره: پوشه رو باز می‌کنی و با یک کلیک **Publish repository** می‌کنی.

## مرحله ۳ — تنظیم اولیه‌ی config.js

۱. توی ریپو برو به مسیر `src/config.js`.
۲. روی آیکون مداد (Edit this file) بالا-راست صفحه بزن.
۳. مقدار `GITHUB_OWNER` رو با یوزرنیم گیت‌هابت و `GITHUB_REPO` رو با اسم ریپو جایگزین کن.
۴. پایین صفحه **Commit changes** بزن. (فعلاً `WORKER_URL` رو دست نزن — توی مرحله‌ی ۷ برمی‌گردیم اینجا.)

## مرحله ۴ — فعال کردن GitHub Pages

۱. توی ریپو برو **Settings → Pages**.
۲. زیر «Build and deployment» → **Source** → گزینه‌ی **GitHub Actions** رو انتخاب کن.
۳. برو تب **Actions** بالای صفحه، صبر کن ورک‌فلوی «Deploy Mini App to GitHub Pages» با علامت سبز تموم بشه (۱-۲ دقیقه).
۴. برگرد به **Settings → Pages** — آدرس سایتت بالای صفحه نشون داده می‌شه، چیزی شبیه:
   `https://USERNAME.github.io/podcast-app/` — این آدرس رو یادداشت کن.

## مرحله ۵ — ساخت بات تلگرام

۱. توی تلگرام به [@BotFather](https://t.me/BotFather) پیام بده.
۲. `/newbot` بزن → یک اسم نمایشی بده → یک یوزرنیم بده (باید به `bot` ختم بشه).
۳. توکنی که می‌ده رو کپی و یه‌جا (مثلاً Notes گوشیت) ذخیره کن — چیزی شبیه `123456:ABC-DEF...`
۴. بات رو ادمین کانال پادکستت کن: تنظیمات کانال → Administrators → Add Admin → بات رو پیدا کن → اضافه کن.

## مرحله ۶ — ساخت پروکسی صوتی (Cloudflare Worker)

۱. به [dash.cloudflare.com](https://dash.cloudflare.com) برو، یک حساب رایگان بساز.
۲. از منوی سمت چپ **Workers & Pages → Create → Create Worker**.
۳. یک اسم بده (مثلاً `podcast-proxy`) → **Deploy**.
۴. **Edit code** بزن، کد پیش‌فرض رو پاک کن، محتوای فایل `worker/index.js` همین پروژه رو کپی-پیست کن → **Deploy**.
۵. تب **Settings** همون Worker → **Variables and Secrets** → یک متغیر مخفی جدید بساز:
   نام: `BOT_TOKEN` — مقدار: همون توکنی که از BotFather گرفتی → ذخیره.
۶. از بالای صفحه آدرس Worker رو کپی کن (شبیه `https://podcast-proxy.YOURNAME.workers.dev`).

## مرحله ۷ — تکمیل config.js

۱. دوباره برو `src/config.js` توی گیت‌هاب → Edit.
۲. `WORKER_URL` رو با آدرس Worker مرحله‌ی قبل پر کن.
۳. **Commit changes** — این بار GitHub Action خودش دوباره اپ رو می‌سازه و آپدیت می‌کنه (چند دقیقه صبر کن).

## مرحله ۸ — وصل کردن وب‌هوک بات به Worker

یک تب جدید مرورگر باز کن و این آدرس رو با مقدارهای خودت پر کن، بعد بازش کن (Enter):

```
https://api.telegram.org/bot<TOKEN>/setWebhook?url=<WORKER_URL>/webhook
```

باید پیامی شامل `"ok":true` ببینی.

## مرحله ۹ — ساخت مینی‌اپ در BotFather

۱. به BotFather پیام بده: `/newapp`
۲. باتت رو انتخاب کن.
۳. عنوان و توضیح کوتاه بده (عکس اختیاریه).
۴. وقتی URL خواست، آدرس GitHub Pages مرحله‌ی ۴ رو بده.
۵. یک shortname هم می‌خواد — هرچی دلت خواست بده. تمام، حالا از طریق منوی بات می‌تونی مینی‌اپ رو باز کنی.

---

## افزودن اولین اپیزود (کار روزمره)

۱. یکی از فایل‌های صوتی کانالت رو **فوروارد** کن به همون بات (داخل چت خصوصی با بات).
۲. بات یه پیام با یک قطعه‌ی JSON برات می‌فرسته — اون متن رو کپی کن.
۳. توی مرورگر برو به: `https://USERNAME.github.io/podcast-app/admin/`
۴. **(بار اول)** یک GitHub Token بساز:
   پروفایل گیت‌هاب → Settings → پایین صفحه **Developer settings** → **Fine-grained tokens** →
   **Generate new token** → Repository access: فقط همین ریپو → Permissions → **Contents: Read and write** →
   Generate token → کپی کن (فقط یه بار نشونش می‌ده!).
۵. توکن رو توی صفحه‌ی ورود پنل ادمین پیست کن و وارد شو.
۶. یک دسته‌بندی بساز (مثلاً «تکنولوژی»).
۷. توی بخش «چسباندن سریع»، JSON مرحله‌ی ۲ رو پیست کن → **تجزیه و پر کردن فرم** رو بزن.
۸. دسته‌بندی رو انتخاب کن، در صورت تمایل توضیحات بنویس → **افزودن اپیزود**.

تغییرات بلافاصله به‌صورت یک کامیت روی گیت‌هاب ذخیره می‌شه و چند ثانیه بعد توی مینی‌اپ هم دیده می‌شه.

## تست نهایی

توی تلگرام برو به بات یا کانالت، روی دکمه‌ی منو (یا لینک مینی‌اپ که با BotFather ساختی) بزن.
مینی‌اپ باز می‌شه، اپیزودت رو می‌بینی، روش بزن تا پخش شه — با پلیر تمام‌صفحه و اسکرابر موج صوتی.

---

## نکات امنیتی

- توکن بات هیچ‌وقت توی کد فرانت‌اند یا ریپازیتوری گیت‌هاب نیست — فقط داخل Cloudflare Worker به‌صورت secret نگه داشته می‌شه.
- توکن گیت‌هاب پنل ادمین (PAT) رو فقط با دسترسی محدود به همین یک ریپو بساز.
- اگه می‌خوای محتوا کاملاً خصوصی بمونه، ریپازیتوری رو Private بگذار (GitHub Pages روی ریپوی پرایوت هم
  با حساب رایگان کار می‌کنه، فقط لینک نهایی عمومی قابل دسترسیه).

## تست لوکال (اختیاری، برای کسی که Node.js نصب داره)

```bash
npm install
npm run dev
```
